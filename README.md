# FLAC Rebuilder

一个用于清理 FLAC 音频文件中非标准/冗余垃圾数据（如尾部非标准 ID3v1/ID3v2 标签等），并重新生成符合规范音频流的工具。支持**命令行批处理**与**现代图形界面（GUI）拖放**操作。

## 功能特性

- **音频流重构重建**：解码 FLAC 内部真实的 PCM 原始声音数据，重新编码并写回，物理级地过滤并丢弃原文件末尾的所有非音频垃圾字节。
- **无损音质校验**：处理前后计算并比对 PCM 的 MD5 值，确保声音数据 **100% 逐比特（bit-for-bit）完全相同**。
- **标签与封面保留**：100% 还原保留原有的元数据标签（歌名、歌手、专辑、歌词等）以及内嵌的专辑封面图片。
- **高精度定位表重建**：在重写文件时自动重新计算并写入标准的定位表（Seek Table/Seek Points），彻底解决播放进度条快进/快退响应缓慢或卡顿的问题。
- **现代 GUI 界面**：支持拖放（Drag & Drop）多个文件和文件夹到窗口中，异步处理任务，界面流畅不卡死。

## 音频 MD5 无损校验说明与局限性

本工具在处理前后会解码出 PCM 数据并计算其 MD5，以确保音频“逐比特（bit-for-bit）完全无损”。但在以下特定情况下，可能会导致处理前后的 PCM MD5 校验失败（从而触发程序的保护性回滚，不写入坏损数据）：

1. **损坏或不完整的 FLAC 文件**
   - 若原 FLAC 文件的音频流已损坏或截断，`libsndfile` 解码器读取到损坏帧时，可能会提前终止或用 0 填充该帧。当用 `soundfile` 重新写回时，生成的是已修复但长度/内容与原损坏流不同的音频流，进而造成 MD5 发生改变。
2. **非标准 Subtype/位深格式与 PCM_32 局限性**
   - 工具对标准 16-bit (`PCM_16`) 和 24-bit (`PCM_24`) FLAC 提供完善支持。
   - **对于 32-bit 整型 (`PCM_32`)**：FLAC 官方标准子集和绝大多数播放器最高只支持到 24-bit 位深。当尝试使用 `PCM_32` 重新编码时，底层库（如 `libsndfile`）可能会将其隐式降级为 24-bit 进行编码。这会导致低 8 位的采样精度在重构时丢失（变为 0），使再次读取时的 PCM 字节数据发生变化，导致 MD5 校验失败。
   - 若音频使用非标准浮点格式（如 `FLOAT_32`）或特殊的压缩模式，重写音频时数值精度可能会因浮点舍入误差或抖动（Dithering）导致二进制数据产生极微小差异，从而改变 MD5。
3. **多声道或非标准声道布局**
   - 含有非常规声道（如 5.1、7.1 声道）的音频，在底层进行声道编解码重构时，其声道顺序、布局元数据可能被重新映射，进而影响重构后读取出的 PCM 数据 MD5。
4. **低 8 位不为 0 的异常 24-bit 数据**
   - 在处理 `PCM_24` 时，底层库会将 24-bit 转换并按高 24 位对齐存入 32-bit 整型（低 8 位通常填充为 0）。工具在计算 MD5 时会过滤低 8 位 `(data >> 8)`。若原音频在低 8 位混入了无效的脏数据或填充数据，重构并重新对齐写入后，这些脏数据将被清理，导致最终读取回的 PCM 数据的二进制内容产生细微变化而改变 MD5。
5. **libsndfile 库版本差异**
   - 不同的操作系统环境或 `libsndfile` 的内部版本，解压某些非常规采样率或非常规压缩块的 FLAC 帧时可能存在差异，这也可能在重构时引发微小的二进制变化。

## 运行环境

- Python 3.7+
- 依赖库：
  - `mutagen`
  - `soundfile` (在 Windows 下自带 `libsndfile` DLL)
  - `numpy`
  - `PyQt6` (仅 GUI 版本需要)

## 安装与运行

### 1. 使用 `uv` 运行 (推荐)
如果您安装了 `uv`，无需手动安装依赖即可运行：

*   **运行命令行版本**：
    ```bash
    uv run flac_rebuilder.py <file1.flac> <file2.flac> <directory_path>
    ```
*   **运行图形界面版本**：
    ```bash
    uv run flac_rebuilder_gui.py
    ```

### 2. 使用标准 Python 运行
首先安装依赖：
```bash
pip install -r requirements.txt
```
然后运行对应的脚本：
```bash
# 运行命令行版
python flac_rebuilder.py [音频文件/文件夹路径...]

# 运行图形界面版
python flac_rebuilder_gui.py
```

## 打包成独立 EXE

项目内置了 `build.bat` 打包脚本。如果您的 Windows 环境上装有 `uv`，直接双击运行 `build.bat` 即可自动编译生成独立可执行程序。

或者，您也可以使用标准 PyInstaller 进行手动打包：

*   **打包命令行版**：
    ```bash
    pyinstaller --onefile --console flac_rebuilder.py
    ```
*   **打包图形界面版**：
    ```bash
    pyinstaller --onefile --noconsole flac_rebuilder_gui.py
    ```

编译完成后，生成的可执行程序位于 `dist/` 目录下。
